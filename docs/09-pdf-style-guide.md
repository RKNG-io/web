## Colour Palette in PDF

### Primary Usage

| Colour | Hex | Use in PDF |
|--------|-----|------------|
| Charcoal | #2d2926 | Cover background, section headers, text |

### Accent Rules

- **Fuchsia**: Action items, CTAs, important numbers, links â€” never backgrounds
- **Mint**: Success states, things they're doing right, the "after" state
- **Blue**: Neutral sections, supporting info, less urgent items

## Tone by Section

| Section | Tone | Example |
|---------|------|---------|
| Cover | Confident, personal | [Their name] â€” no preamble |
| Mirror | Warm, listening | "You told us..." |
| Snapshot | Clear-eyed, fair | "What's working... what's missing" |
| Diagnosis | Insightful, direct | Reframe or give permission |

### Tone Rules

**Never:**
- "The real problem isn't X. It's Y." â€” patronising
- "What you need to understand is..." â€” condescending
- "You might not realise that..." â€” assumes ignorance
- "Let me explain..." â€” positions us as teacher
- Explaining back to them what they just told us

**Instead:**
- Reframe: "The gap is smaller than it feels."
- Give permission: "You've been waiting to feel ready. You already are."
- Validate: "That stuck feeling makes sense."
- Be specific: Use their numbers, their words, their situation

## Generating the PDF

### Technical Approach

**Option 1: React-PDF**
- Generate in-browser or via API
- Full control over layout
- Use `@react-pdf/renderer`

**Option 2: HTML â†’ PDF Service**
- Render HTML template, convert via Puppeteer or PDFShift
- Easier to style, less precise control
- Faster to iterate

**Option 3: Hybrid**
- Generate web version, offer "Download PDF" via API
- Web version is primary, PDF is artifact

### Recommended: Option 3

```
1. Generate Reckoning as structured JSON
2. Render on web at /reckoning/[id]
3. "Download PDF" button calls /api/pdf/[id]
4. API renders same data into PDF template
5. Return PDF with filename: "[Name]-Reckoning-[Date].pdf"
```
